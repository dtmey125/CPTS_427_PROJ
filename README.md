# CPTS_427_PROJ

This project creates a network of 4 Ubuntu virtual machines with Vagrant (hosted on KVM), and has several scripts to inject various network faults using Ansible, which can be observed through the Nagios web interface at:
http://192.168.50.11/nagios4/

Reports of the project are in the `report` directory, with various screenshots showing the states before, during, and after the fault injection.

**Themes used:** Cloud infrastructure components, Secure scaling and deployment, Fault tolerance, Distributed system security

**More info (Design decisions, trade-offs, lessons learned) is in the file`report/Project Summary and Analysis.pdf`**

## Building and Running

This project was built in Fedora Linux 43, with [KVM](https://linux-kvm.org/page/Main_Page), [libvirt](https://libvirt.org/), [Vagrant](https://developer.hashicorp.com/vagrant), and [Ansible](https://docs.ansible.com/) installed. Performance and compatibility may vary across distributions, and is not supported on Windows.

First, mark all the utility scripts as executable:

```
chmod +x create.sh destroy.sh down.sh inject_faults.sh
ansible_inventory.sh
```

To create the virtual machines, enter the `src` directory and run:

```
./create.sh
```

The initial creation will take a long time, especially while Vagrant downloads the appropriate OS images (in this case, Ubuntu 22.04).

### Utility Scripts included:

- `create.sh` - Boots up or creates the VMs defined in `src/nodes.yml`.
- `down.sh` - Shuts down all nodes.
- `destroy.sh` - Deletes all virtual machine instances.
- `ansible_inventory.sh` - Lists the Ansible inventory generated by Vagrant.
- `inject_faults.sh` - Injects faults into the hosted nodes.
- `recover_faults.sh` - Reverses all fault injections
